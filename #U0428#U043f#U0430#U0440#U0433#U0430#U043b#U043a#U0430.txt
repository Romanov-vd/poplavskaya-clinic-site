========================================
ШПАРГАЛКА ПО КУРСОВОМУ ПРОЕКТУ
========================================

--- 1. ЧТО ЭТО ЗА ПРОЕКТ ---

Это сайт на Django (Python-фреймворк) для вымышленной клиники.
Состоит из 3 страниц: Главная, Услуги, Контакты.
Данные (услуги, цены, адрес) прописаны прямо в коде — база данных не используется.


--- 2. СТРУКТУРА ПАПОК И ФАЙЛОВ (что за что отвечает) ---

Папка проекта:

  manage.py — главный файл. Через него запускают сервер, делают миграции и т.д.
  requirements.txt — список того, что нужно установить (Django).
  db.sqlite3 — файл базы данных. Пока пустой.

Папка clinic/ — настройки всего сайта:

  settings.py — настройки: язык (русский), время (Москва), где лежат шаблоны и статика.
  urls.py — таблица маршрутов: какой адрес ведёт на какую страницу.
  views.py — функции, которые готовят данные и отправляют их в шаблоны.
  asgi.py и wsgi.py — для запуска на сервере. В курсовой не трогаем.

Папка templates/ — HTML-шаблоны (то, что видит пользователь):

  base.html — каркас сайта (шапка, меню, подвал). На нём основаны все страницы.
  index.html — главная страница с приветствием.
  services.html — страница «Услуги и цены».
  contacts.html — страница «Контакты».

Папка static/clinic/images/ — картинки:

  medcenter.jpg — фото клиники на главной.
  map.png — схема проезда на странице контактов.


--- 3. КАК ЭТО РАБОТАЕТ (ПРОСТЫМИ СЛОВАМИ) ---

1. Пользователь открывает сайт (например, http://127.0.0.1:8000/).
2. Django смотрит в файл urls.py и понимает, какая функция из views.py должна сработать.
3. Функция в views.py собирает данные (текст, список услуг и т.д.) в словарь context.
4. Эти данные отправляются в HTML-шаблон (например, index.html).
5. Шаблон подставляет данные в нужные места и показывает готовую страницу.

Маршруты (как Django понимает, какую страницу показать):

  / → index() → index.html (Главная)
  /services/ → services() → services.html (Услуги)
  /contacts/ → contacts() → contacts.html (Контакты)


--- 4. СЛОЖНЫЕ МЕСТА В КОДЕ (что спросит преподаватель) ---

4.1. Наследование шаблонов

  В base.html есть «дырки» — {% block content %} и {% block title %}.
  Другие страницы (index.html, services.html) просто заполняют эти дырки.
  Зачем: чтобы не копировать шапку и меню в каждую страницу.

  Вопрос: «Почему не скопировать код в каждый файл?»
  Ответ: если нужно поменять шапку сайта, меняем только base.html,
  а не все 3 страницы по отдельности.

4.2. Подсветка активной вкладки в меню

  В base.html написано: если active_page == 'index', то ссылке даётся класс active.
  В каждой функции views.py написано, какая страница сейчас открыта.
  Зачем: чтобы кнопка в меню подсвечивалась, когда мы на этой странице.

4.3. Картинки

  Картинки лежат в папке static/, а не в templates/.
  В шаблонах их подключают через {% static 'путь/к/файлу' %}.
  Вопрос: «Почему картинки не в той же папке, что HTML?»
  Ответ: Django разделяет код (шаблоны) и контент (картинки, CSS).
  Это правильный подход для любого сайта.



--- 5. КАК ЗАПУСТИТЬ (пошагово) ---

1. Установить Python (если нет).
2. Открыть командную строку (терминал).
3. Установить Django одной командой:

     pip install django

4. Перейти в папку с проектом:

     cd "Сайт Лера Гармония Здоровья"

5. Запустить сервер:

     python manage.py runserver

6. Открыть в браузере: http://127.0.0.1:8000/

Готово! Сайт работает.

Чтобы остановить сервер — нажать Ctrl + C.


--- 6. ВОПРОСЫ ПРЕПОДАВАТЕЛЯ И ОТВЕТЫ (коротко) ---

В: Какая архитектура используется?
О: MVT — Model-View-Template (Django-версия MVC).
   Моделей нет (нет БД), View — функции в views.py, Template — HTML.

В: Почему Django, а не Flask?
О: Django даёт всё готовое: маршруты, шаблоны, админку, работу с БД.
   Flask — это «конструктор», нужно самому подключать детали.

В: Как добавить новую страницу?
О: 1) Создать HTML в templates/.
   2) Написать функцию в views.py.
   3) Добавить маршрут в urls.py.

В: Как работает render()?
О: Берёт шаблон, вставляет в него данные из словаря, возвращает готовый HTML.

В: Почему CSS встроен в base.html?
О: Для упрощения. В большом проекте CSS выносят в отдельный файл.

В: Как сделать форму обратной связи?
О: Создать модель Feedback, форму, view с обработкой POST,
   и в шаблоне добавить <form method="post"> с {% csrf_token %}.

В: Что такое CSRF-токен?
О: Защита от взлома. Django проверяет, что запрос пришёл с вашего сайта,
   а не с левого.
